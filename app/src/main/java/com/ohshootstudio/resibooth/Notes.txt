OH Shoot! — Android Photobooth App Specification
TECH STACK

Language: Kotlin
UI: Jetpack Compose (BOM 2024.02+)
Camera: CameraX 1.3+
Navigation: Navigation Compose 2.7+
Dependency Injection: Hilt 2.50+

State: ViewModel + StateFlow

SESSIONVIEWMODEL STATE

selectedLayout: Int — number of photos (1, 2, 3, or 4)
capturedPhotos: List<Bitmap?> — null = not yet captured, size equals selectedLayout
currentShotIndex: Int — which shot is next
copyCount: Int — clamped 1..5
sessionComplete: Boolean
appSettings: AppSettings — observed as StateFlow from DataStore

SESSIONVIEWMODEL ACTIONS

startSession(layout: Int) — reset all state, set layout, clear photos
savePhoto(index: Int, bitmap: Bitmap) — store at index
retakePhoto(index: Int) — null out that slot
retakeAll() — null all slots, reset index to 0
setCopyCount(n: Int) — clamp to 1..5
resetSession() — full reset before returning to Standby

CAMERA — CameraManager

Request CAMERA permission at runtime before navigating to Capture
Bind CameraX Preview + ImageCapture use cases to front camera by default
Expose selectCamera(facing: CameraSelector) to switch cameras
Mirror: scaleX(-1f) modifier on PreviewView wrapper
Grayscale filter: ColorMatrix with greyscale coefficients applied via Paint on preview
Capture output: JPEG via ImageCapture in-memory (no file write)
Target resolution: 1280x960 or best available
Countdown runs in coroutine: delay(1000) three times updating countdownValue StateFlow, then capture
On capture success: decode to Bitmap, call viewModel.savePhoto(index, bitmap)


PRINTING — UsbPrinterManager

Use Android USB Host API to send raw ESC/POS bytes directly to printer
No print dialog, no PDF, no system print framework
Target printer: Xprinter XP-N160II, 80mm roll, 203 DPI, USB OTG

USB SETUP

Declare uses-feature: android.hardware.usb.host in AndroidManifest
Create res/xml/usb_device_filter.xml filtering vendor-id="4070" (0x0FE6) and class="7" (printer class)
Add intent-filter for ACTION_USB_DEVICE_ATTACHED on MainActivity pointing to that filter file
Register BroadcastReceiver for ACTION_USB_DEVICE_ATTACHED and ACTION_USB_DEVICE_DETACHED
On attach: enumerate USB devices, match by VID 0x0FE6 or device class 7
Request USB permission via PendingIntent before opening
Expose printerState: StateFlow<PrinterState> — sealed class with Disconnected, Connecting, Ready, Printing, Error(message)

ESCPOS COMMAND BUILDER CLASS

init() → bytes 0x1B 0x40
center() → bytes 0x1B 0x61 0x01
left() → bytes 0x1B 0x61 0x00
bold(on: Boolean) → 0x1B 0x45 then 0x01 or 0x00
text(str: String) → UTF-8 encoded bytes
newline(n: Int) → 0x0A repeated n times
cutPaper() → bytes 0x1D 0x56 0x42 0x00
printBitmap(bmp: Bitmap) → see raster protocol below
feedAndCut() → newline(4) + cutPaper()

BITMAP RASTER PROTOCOL (GS v 0)

Resize dithered bitmap to exactly 576 pixels wide (80mm at 203 DPI minus margins)
Pad height to nearest multiple of 8 with white rows if needed
Command header: 0x1D 0x76 0x30 0x00, then xL xH where width-in-bytes = 576/8 = 72, then yL yH as little-endian 16-bit pixel height
For each row: pack 8 pixels into 1 byte MSB first, black=1 white=0
Append all row bytes after header
Write to USB bulk-out endpoint in chunks of max 16384 bytes

IMAGE PROCESSING PIPELINE (run in this exact order before printing)

Step 1: decode captured Bitmap
Step 2: scale to 576px wide, preserve aspect ratio
Step 3: convert to greyscale — luminance = 0.299R + 0.587G + 0.114B per pixel
Step 4: contrast boost — value = clamp((lum - 128) × 1.4 + 128, 0, 255)
Step 5: Floyd-Steinberg dithering — for each pixel left-to-right top-to-bottom: quantize to 0 or 255, distribute error to right neighbor (×7/16), bottom-left (×3/16), bottom (×5/16), bottom-right (×1/16)
Step 6: result Bitmap must be ARGB_8888 where every pixel is either 0xFF000000 or 0xFFFFFFFF

FULL PRINT JOB SEQUENCE

EscPos.init()
EscPos.center()
EscPos.bold(true) + EscPos.text(headerText) + EscPos.bold(false) + EscPos.newline(1)
For each photo: run image pipeline, EscPos.printBitmap(bmp), EscPos.newline(1)
EscPos.text(footerText) + EscPos.newline(1)
EscPos.feedAndCut()
Repeat above for each additional copy if copyCount > 1
Write all bytes to USB bulk-out endpoint

ERROR HANDLING

Camera permission denied: full-screen message with "Open Settings" deep-link, no crash
Camera in use: error message on Capture screen with retry button
No front camera: auto-fallback to rear
No printer connected: show status in settings dot, show toast on Print tap, still navigate to Done
Printer disconnects mid-print: catch IOException, show error snackbar on Done screen
USB permission denied: rationale dialog with retry
Print timeout: progress indicator, timeout after 30 seconds
App backgrounded mid-session: ViewModel survives configuration changes, session resumes
Retake mid-session: navigating back to Capture must pre-select the correct slot index
